## 任务验收

### AUTH-001: DB Schema
- `api_keys` 表字段完整（id, userId unique, encryptedKey, encryptedSecret, binanceUid, isValid, timestamps）
- `ENCRYPTION_KEY` 类型定义已添加（z.string().length(64)）
- `pnpm db:push` 需要真实 DB，用户需在本地配置 DATABASE_URL 后执行

### AUTH-002: tRPC Router
- `getStatus`: protectedProcedure，返回 `{ bound, uid? }` 不暴露加密字段
- `bind`: HMAC-SHA256 Binance 签名验证 → canTrade 检查 → AES-256-GCM 加密 → upsert
- `unbind`: 软删除（delete from api_keys where userId = current）
- 错误均通过 TRPCError 返回，不抛 500

### AUTH-003: 前端登录/注册
- `/login`, `/register` 添加 `beforeLoad` 已登录重定向
- SignInForm: Card 布局 + trade-mind 品牌 + Alert 错误展示
- SignUpForm: 新增 confirmPassword 字段 + refine 校验 + 同样 Card 布局

### AUTH-004: 设置页
- settings.tsx 实现完整绑定/解绑 UI
- `queryClient.invalidateQueries` 触发 AppLayout 重新获取 `apiKeyBound` 状态
- AppLayout 接入 `trpc.apiKey.getStatus`（`enabled: isLoggedIn`，避免未登录 UNAUTHORIZED）
- 全局横幅在绑定成功后通过 query invalidation 自动消失
