# SweetWave 产品简报 — trade-mind

> 版本：INIT｜生成日期：2026-06-19

---

## 1. 一句话想法

为个人加密货币交易者打造一个覆盖"行情调研 → 交易决策 → 自动下单 → 复盘分析"完整闭环的 AI 交易助手，帮助用户在每一个交易阶段都做出更理性的决策。

---

## 2. 目标用户

- **主要用户**：有一定加密货币交易经验的个人投资者（非机构），主要活跃在 Binance 平台
- **行为特征**：自主下单、关注板块轮动、持续跟踪热点、需要事后复盘
- **当前痛点来源**：信息分散、情绪化交易、缺乏系统性复盘机制

---

## 3. 用户痛点

| # | 痛点 | 当前解法 | 问题 |
|---|---|---|---|
| P1 | 不知道当前哪个板块最热、龙头是谁 | 手动刷推特/社群 | 费时、信息质量参差不齐 |
| P2 | 宏观事件影响加密市场，难以及时感知 | 零散新闻 | 反应滞后、缺乏聚焦 |
| P3 | Binance Alpha 项目信息分散，难以系统跟踪 | 手动查看 | 错过底部建仓时机 |
| P4 | 已平仓订单没有系统复盘 | 手动整理 Excel | 效率低、复盘深度不够 |
| P5 | 连续亏损/连续盈利时没有外部冷却机制 | 靠自律 | 情绪化加仓/扩张 |

---

## 4. 核心场景

### S1：板块轮动感知
用户打开产品，首页即可看到当前 AI 汇总的热门板块排名和各板块龙头币种，快速定位交易机会。

### S2：宏观消息追踪
产品自动收集并展示影响全球金融/加密市场的突发事件和热点新闻（战争、央行政策、监管动态等），以简报形式推送给用户。

### S3：Binance Alpha 定投机会
系统定时爬取 Binance Alpha 项目列表，识别处于底部长期盘整的项目，基于"Alpha 项目必然有流动性退出拉盘"策略，标记具有定投价值的候选。

### S4：订单自动复盘
用户授权连接 Binance API，系统拉取已平仓订单，结合用户填写的"开仓逻辑/平仓逻辑"，由 AI 生成每笔订单的复盘诊断报告。

### S5：资金曲线分析与风险预警
系统持续追踪用户账户的盈亏曲线，在检测到连续亏损或连续盈利异常时，主动触发 AI 冷静复盘提示，干预非理性操作冲动。

---

## 5. MVP 边界

MVP 聚焦以下 **最小可用闭环**：

| 模块 | MVP 范围 |
|---|---|
| 行情调研 | 板块轮动（热门板块 + 龙头展示）+ 宏观消息简报 |
| Alpha 跟踪 | Binance Alpha 项目列表展示 + 底部盘整标记 |
| 订单复盘 | 接入 Binance API 拉取已平仓订单 + AI 复盘报告生成（用户可输入逻辑） |
| 资金曲线 | 账户盈亏曲线可视化 + 连续亏损/盈利风险预警 |

> **自动下单**暂不纳入 MVP，降低风险和开发复杂度。

---

## 6. 不做什么

- **不做自动下单**（MVP 阶段，降低风险）
- **不做社交/跟单功能**（保持 toC 个人助手定位）
- **不做多交易所接入**（MVP 只接 Binance）
- **不做量化策略回测**（超出 MVP 范围）
- **不做移动端 App**（先做 Web）
- **不做 KYC / 合规模块**（MVP 阶段不处理资金托管）

---

## 7. 成功指标

| 指标 | MVP 目标 |
|---|---|
| 用户连接 Binance API 并完成首次复盘 | 核心转化路径跑通 |
| 订单复盘报告生成成功率 | ≥ 90% |
| 用户每周主动查看行情调研 | 体现信息价值留存 |
| 资金曲线风险预警触发后用户查看率 | 体现预警有效性 |

## 8. 待确认问题

| # | 问题 | 影响 |
|---|---|---|
| Q1 | Binance Alpha 项目数据来源：官方 API 还是页面抓取？ | 影响数据稳定性和合规性 |
| Q2 | 宏观消息的信息源：接第三方 API（如 NewsAPI、CryptoPanic）还是自建爬虫？ | 影响数据成本和质量 |
| Q3 | Binance API 接入方式：只读（只拉取订单）还是读写（支持未来自动下单）？ | 影响安全策略 |
| Q4 | "连续亏损/盈利"的触发阈值如何定义？（如：连续 3 笔亏损） | 影响风险预警灵敏度 |
| Q5 | 产品是否需要用户账号体系？（数据是否需要云端存储同步） | 影响数据架构 |
