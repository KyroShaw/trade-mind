---

## 1. 产品目标

为个人加密货币交易者构建覆盖"行情调研 → 交易决策 → 复盘分析"完整闭环的 AI 交易助手，帮助用户减少信息噪音、提高决策理性、建立系统性复盘习惯。

**MVP 核心指标**
- 用户连接 Binance API 并完成首次订单复盘（核心转化路径跑通）
- 订单复盘报告生成成功率 ≥ 90%

---

## 2. 目标用户

- 有一定加密货币交易经验的**个人投资者**（非机构）
- 主要活跃在 **Binance** 平台
- 自主下单，关注板块轮动，需要事后复盘

---

## 3. 功能范围

### F-001：用户账号与 Binance API 绑定

> 用户需要账号来持久化 API Key 和复盘数据。

| ID | 功能描述 |
|---|---|
| F-001-1 | 用户注册/登录（邮箱密码） |
| F-001-2 | 绑定 Binance API Key（只读权限） |
| F-001-3 | API Key 加密存储，前端不可见 |

**验收标准**

- [AC-001-1] 用户可完成注册并登录
- [AC-001-2] 绑定 API Key 后系统可成功拉取 Binance 账户基本信息
- [AC-001-3] API Key 不以明文形式出现在任何响应或日志中

---

### F-002：板块轮动感知

> 展示当前加密市场热门板块及板块龙头，帮助用户快速定位交易机会。

| ID | 功能描述 |
|---|---|
| F-002-1 | 展示当前热门板块列表（AI 汇总）及热度排名 |
| F-002-2 | 每个板块展示其龙头币种（价格、涨跌幅） |
| F-002-3 | 数据定时刷新（每 30 分钟） |

**验收标准**

- [AC-002-1] 首页展示至少 5 个热门板块，每板块至少 1 个龙头
- [AC-002-2] 数据标注更新时间
- [AC-002-3] 刷新周期 ≤ 30 分钟

---

### F-003：宏观消息追踪

> 收集并展示影响全球金融/加密市场的热点事件，以简报形式呈现。

| ID | 功能描述 |
|---|---|
| F-003-1 | 接入第三方新闻 API（如 CryptoPanic）收集热点消息 |
| F-003-2 | AI 对消息进行加密市场影响摘要（一句话） |
| F-003-3 | 按时间倒序展示，支持标签筛选（宏观/监管/市场） |

**验收标准**

- [AC-003-1] 每日至少展示 10 条有效消息
- [AC-003-2] 每条消息附带 AI 影响摘要
- [AC-003-3] 支持按标签筛选

---

### F-004：Binance Alpha 项目跟踪

> 定时收集 Binance Alpha 项目信息，识别底部盘整候选，辅助定投决策。

| ID | 功能描述 |
|---|---|
| F-004-1 | 定时抓取 Binance Alpha 项目列表（每日一次） |
| F-004-2 | 展示项目基本信息（名称、价格、7/30 日涨跌幅） |
| F-004-3 | 自动标记处于"底部盘整"状态的项目（30 日跌幅 > 30% 且近 7 日波动 < 10%） |
| F-004-4 | 标记候选项目为"定投关注" |

**验收标准**

- [AC-004-1] 项目列表每日自动更新
- [AC-004-2] 底部盘整标记规则透明可见（展示判断依据）
- [AC-004-3] 用户可手动添加/移除"定投关注"标记

---

### F-005：已平仓订单复盘

> 拉取 Binance 已平仓订单，结合用户输入的交易逻辑，由 AI 生成复盘诊断报告。

| ID | 功能描述 |
|---|---|
| F-005-1 | 通过 Binance API 拉取近 90 天已平仓订单 |
| F-005-2 | 展示订单列表（交易对、方向、开仓价、平仓价、盈亏） |
| F-005-3 | 用户可为每笔订单填写"开仓逻辑"和"平仓逻辑" |
| F-005-4 | AI 结合用户逻辑生成复盘诊断报告（优点、问题、建议） |
| F-005-5 | 复盘报告可导出为 Markdown |

**验收标准**

- [AC-005-1] 成功拉取近 90 天订单，数量与 Binance 一致
- [AC-005-2] 用户提交逻辑后 30 秒内生成复盘报告
- [AC-005-3] 复盘报告包含至少 3 个维度（执行质量、风险控制、改进建议）
- [AC-005-4] 报告可一键复制/导出

---

### F-006：资金曲线分析与风险预警

> 可视化用户账户盈亏曲线，在检测到异常连续亏损/盈利时触发 AI 冷静提示。

| ID | 功能描述 |
|---|---|
| F-006-1 | 基于已平仓订单计算并展示账户盈亏曲线（按时间） |
| F-006-2 | 检测连续亏损（默认 3 笔）或连续盈利（默认 5 笔）触发预警 |
| F-006-3 | 预警触发时展示 AI 冷静复盘提示（分析近期交易模式） |
| F-006-4 | 用户可自定义预警阈值 |

**验收标准**

- [AC-006-1] 曲线按天/周可切换视图
- [AC-006-2] 连续亏损 3 笔后在页面顶部展示预警横幅
- [AC-006-3] 预警信息包含近期交易模式分析和操作建议
- [AC-006-4] 用户可在设置中调整触发阈值（2-10 笔）

---

## 4. 非功能需求

| 类别 | 要求 |
|---|---|
| 安全 | Binance API Key 加密存储，最小权限原则（只读） |
| 响应时间 | 普通页面加载 < 2s；AI 报告生成 < 30s |
| 数据刷新 | 行情类数据 ≤ 30 分钟；Alpha 项目 ≤ 24 小时 |
| 错误处理 | API 拉取失败时展示友好提示和上次更新时间 |

---

## 5. 不在范围内

- 自动下单
- 社交/跟单功能
- 多交易所接入
- 量化策略回测
- 移动端 App
- KYC / 合规模块
