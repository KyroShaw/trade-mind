# DESIGN: order-review — 订单复盘

> 模块：order-review | PRD：F-005 | 状态：READY

---

## 页面地图

```
/orders          订单列表页
/orders/:id      订单详情 + 复盘报告页
```

---

## 主流程

### 查看订单列表
1. 用户进入 `/orders`，系统自动拉取近 90 天已平仓订单
2. 展示订单列表（交易对、方向、开仓价、平仓价、盈亏金额/比率）
3. 支持按交易对、时间范围、盈亏状态筛选

### 订单复盘流程
1. 用户点击某条订单 → 进入 `/orders/:id`
2. 展示订单详情（完整时间线：开仓 → 持仓 → 平仓）
3. 用户在文本框填写"开仓逻辑"和"平仓逻辑"
4. 点击"生成复盘" → AI 生成诊断报告
5. 报告展示 3 个维度：执行质量、风险控制、改进建议
6. 用户可复制或导出 Markdown

---

## 分支与异常

| 场景 | 处理 |
|---|---|
| Binance API 拉取失败 | 展示上次同步时间，提示重试 |
| 未填写任何逻辑直接生成 | AI 基于订单数据生成通用复盘（不强制填写） |
| 报告生成超时（> 30s） | 展示超时提示，允许重试 |
| 无已平仓订单 | 展示空状态引导（"暂无已平仓订单，开始你的第一笔交易"） |
| 未绑定 API Key | 展示绑定引导，屏蔽列表 |

---

## 交互状态

- 拉取订单中：列表骨架屏
- 生成复盘中：报告区域 loading + 进度文案（"AI 正在分析你的交易..."）
- 报告已生成：可展开/折叠各维度
- 已有复盘记录的订单：在列表中展示"已复盘"标记

---

## 数据说明

- 首次进入拉取近 90 天，后续增量同步
- 复盘逻辑文本和报告内容存储在本地账户体系中
