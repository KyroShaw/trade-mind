# DESIGN: auth — 账号与 API 绑定

> 模块：auth | PRD：F-001 | 状态：READY

---

## 页面地图

```
/login       登录页
/register    注册页
/settings    设置页（包含 API Key 绑定）
```

---

## 主流程

### 注册流程
1. 用户访问 `/register`，填写邮箱 + 密码
2. 提交后收到验证邮件（[假设] MVP 暂不强制邮件验证，直接注册成功）
3. 注册成功 → 跳转至 `/settings` 引导绑定 Binance API Key

### 登录流程
1. 用户访问 `/login`，填写邮箱 + 密码
2. 登录成功 → 跳转至首页（行情调研 `/`）
3. 未绑定 API Key 时，顶部展示引导横幅 "绑定 Binance API 以解锁完整功能"

### API Key 绑定流程
1. 用户进入 `/settings`，填写 Binance API Key（只读权限）
2. 提交后系统校验 Key 有效性（拉取账户基本信息）
3. 校验成功 → 保存加密存储，展示绑定成功提示
4. 校验失败 → 展示错误说明（Key 无效 / 权限不足 / 网络错误）

---

## 分支与异常

| 场景 | 处理 |
|---|---|
| 邮箱已注册 | 注册页提示"邮箱已存在"，引导至登录 |
| 密码错误 | 登录页提示"邮箱或密码不正确"（不区分） |
| API Key 无效 | 绑定页展示错误码和 Binance 帮助链接 |
| API Key 权限不足（非只读） | 提示需要只读权限，并说明如何在 Binance 设置 |
| 未登录访问受保护页 | 重定向至 `/login`，登录后返回原页面 |

---

## 交互状态

- 表单提交中：按钮 loading，禁止重复提交
- API Key 校验中：展示校验进度提示
- 绑定成功：展示绑定的账户名和 UID（脱敏）

---

## 跨模块 UX 依赖

- 所有业务模块页面：未绑定 API Key 时屏蔽数据区域，展示绑定引导
- app-shell：顶部导航展示用户头像/登出入口，未登录时展示登录按钮
